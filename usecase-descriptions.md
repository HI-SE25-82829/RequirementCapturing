#### Use case:

### 1. 대여소 검색 및 조회
|Actor Action | System Response |
|---|---|
|1. 회원이 로그인을 한다. | 2. 등록된 대여소 목록과 검색창을 보여준다. |
|3. 회원은 검색창에 대여소 이름을 입력한다. | 4. 조건에 맞는 대여소 리스트를 보여준다. |


### 2. 대여소 등록
|Actor Action | System Response |
|---|---|
|1. 관리자가 관리자 권한으로 로그인을 한다.  | 2. "대여소 등록하기" 버튼을 보여준다. |
|3. 관리자가 "대여소 등록하기"를 선택한다. | 4. 대여소 등록 입력 폼을 보여준다. 대여소 이름, 위치(도시, 주소), 자전거 보관 가능 수량, 운영 시간 등의 항목이 있다. |
|5. 관리자는 입력 폼에 맞춰 대여소 정보를 입력하고 "등록하기" 버튼을 클릭한다. | 6. 입력값 유효성을 검증에 따른 결과를 표시한다. 유효하다면 등록이 완료되었다는 상태를 표시한다. |


### 3. 대여소 조회
|Actor Action | System Response |
|---|---|
|1. 관리자가 관리자 권한으로 로그인을 한다.  | 2. 등록된 대여소 목록을 보여준다. |


### 4. (대여소) 상세내용 확인
|Actor Action | System Response |
|---|---|
|1. 관리자가 관리자 권한으로 로그인을 한다.  | 2. 등록된 대여소 목록을 보여준다. |
|3. 관리자가 등록된 대여소 목록에서 원하는 대여소 항목의 "상세내용 확인하기" 버튼을 클릭할 수 있다. | 4. 대여소 등록 시 입력한 상세내용을 볼 수 있다. |


### 5. (대여소) 항목 삭제
|Actor Action | System Response |
|---|---|
|1. 관리자가 관리자 권한으로 로그인을 한다.  | 2. 등록된 대여소 목록을 보여준다. |
|3. 관리자가 등록된 대여소 목록에서 원하는 대여소 항목의 "삭제하기" 버튼을 클릭할 수 있다. | 4. "정말로 삭제하시겠습니까?"라는 메시지와 버튼 표시로 삭제 의사를 한번 더 확인한다. |
|5. 관리자가 "정말로 삭제하시겠습니까?" 아래 버튼을 클릭하여 최종적인 삭제 요청을 할 수 있다. | 6. 해당 대여소를 삭제한다.

#### 1. 대여소 상세정보 조회
|Actor Action | System Response |
|---|---|
|1. NONE |2. 대여소 리스트 화면을 출력한다.|
|3. 사용자가 특정 대여소를 선택한다.|4. 대여소의 상세정보(이름, 위치, 자전거 목록)를 보여준다. |


#### 2. 자전거 즉시 대여
|Actor Action | System Response |
|---|---|
|1. NONE |2. 대여소 상세정보 화면을 출력한다.|
|3. 사용자가 즉시 대여 버튼을 클릭한다.|4. 자전거를 할당하고 문자 알림을 전송한다.|

#### 3. 자전거 예약대기 신청
|Actor Action | System Response |
|---|---|
|1. NONE |2. 대여소 상세정보 화면에서 ‘자전거 없음’ 상태를 보여준다.|
|3. 사용자가 예약대기 신청 버튼을 클릭한다.|4. 예약대기를 등록하고 문자 알림을 전송한다.|

#### 4. 문자 알림 전송
|Actor Action | System Response |
|---|---|
|1. NONE |2. 대여 또는 예약 요청 처리 화면을 출력한다.|
|3. 사용자가 요청을 완료한다.|4. 요청 결과에 따라 문자 알림을 전송한다.|

#### 5. 대여 중 자전거 조회
|Actor Action | System Response |
|---|---|
|1. NONE |2. ‘대여 중 자전거 조회’ 화면을 출력한다.|
|3.	사용자가 정보를 확인한다.|4. 대여 중 자전거 정보(대여소 이름, 위치, ID, 제품명, 유형)를 보여준다.|

#### 6. 예약대기 정보 조회
|Actor Action | System Response |
|---|---|
|1. NONE |2. 예약대기 정보 조회 화면을 출력한다.|
|3. 사용자가 예약 정보를 선택한다.|4. 예약 상세정보(대여소 이름, 위치, 자전거 ID, 제품명, 유형)를 보여준다.|

#### 7. 예약대기 취소
|Actor Action | System Response |
|---|---|
|1. NONE |2. 	예약대기 리스트 화면을 출력한다.|
|3. 사용자가 특정 예약을 선택하고 ‘취소’ 버튼을 클릭한다.|4. 예약이 취소되고 완료 메시지를 출력한다.|
