#### Use case:

#### 1. 회원 가입
|Actor Action | System Response |
|---          |---              |
|1. 비회원이 "회원가입" 버튼을 클릭한다. |2. 회원 가입 입력 폼을 화면을 표시한다. |
|3. 회원 정보(ID, 비밀번호, 전화번호, 결제 수단, 선호 자전거 유형(일반/전기) 등)를 입력 후 "회원가입" 버튼을 클릭할 수 있다. |4. 회원 권한을 부여한다. |

#### 2. 회원 탈퇴
|Actor Action | System Response |
|---          |---              |
|1. 회원이 "회원탈퇴" 버튼을 클릭한다. |2. 회원 권한 박탈 및 해당 회원의 데이터를 삭제한다. |

#### 3. 로그인
|Actor Action | System Response |
|---          |---              |
|1. 관리자와 회원이 ID와 비밀번호 입력 후 "로그인" 버튼을 클릭한다. |2. 저장된 ID, 비밀번호와 일치할 경우 접근 권한을 부여하고 로그인 상태의 메인 화면으로 전환한다. |

#### 4. 로그아웃
|Actor Action | System Response |
|---          |---              |
|1. 로그인한 관리자와 회원이 "로그아웃" 버튼을 클릭한다. |2. 비로그인 상태의 메인 화면으로 전환한다. |

#### 5. 대여소 등록
|Actor Action | System Response |
|---|---|
|1. 관리자가 "대여소 등록하기"를 클릭한다. | 2. 대여소 등록 입력 폼을 보여준다. 대여소 이름, 위치(도시, 주소), 자전거 보관 가능 수량, 운영 시간 등의 항목이 있다. |
|3. 관리자는 입력 폼에 맞춰 대여소 정보를 입력하고 "등록하기" 버튼을 클릭한다. | 4. 입력값 유효성을 검증에 따른 결과를 표시한다. 유효하다면 등록이 완료되었다는 상태를 표시한다. |

#### 6. 관리자_대여소 조회 
|Actor Action | System Response |
|---|---|
|1. 관리자가 "대여소 조회"를 클릭한다.  | 2. 등록된 대여소 목록을 보여준다. |

#### 7. 대여소 상세내용 확인
|Actor Action | System Response |
|---|---|
|1. 관리자가 "대여소 조회"를 클릭한다.  | 2. 등록된 대여소 목록을 보여준다. |
|3. 관리자가 등록된 대여소 목록에서 원하는 대여소 항목의 "상세내용 확인하기" 버튼을 클릭할 수 있다. | 4. 대여소 등록 시 입력한 상세내용을 볼 수 있다. |

#### 8. 대여소 항목 삭제
|Actor Action | System Response |
|---|---|
|1 관리자가 "대여소 조회"를 클릭한다.  | 2. 등록된 대여소 목록을 보여준다. |
|3. 관리자가 등록된 대여소 목록에서 원하는 대여소 항목의 "삭제하기" 버튼을 클릭할 수 있다. | 4. "정말로 삭제하시겠습니까?"라는 메시지와 버튼 표시로 삭제 의사를 한번 더 확인한다. |
|5. 관리자가 "정말로 삭제하시겠습니까?" 아래 버튼을 클릭하여 최종적인 삭제 요청을 할 수 있다. | 6. 해당 대여소를 삭제한다.

|---|---|

#### 16. 자전거 예약대기 신청
|Actor Action | System Response |
|---|---|
|1. 회원이 로그인을 하고 대여소 검색 및 조회 화면으로 이동한다. | 2. 등록된 대여소 목록과 검색창을 보여준다. |

#### 20. 자전거 반납
